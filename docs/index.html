Article Study Notes

A portable Codex skill for turning substantial texts into durable Markdown study notes with the right structure for each material type.

Markdown Obsidian Typora GitHub VS Code
01 / 08
The ProblemGeneric summaries fail quietly

Most notes flatten the text.

Paragraph summaries are easy to make and hard to study from.

Lost logic

Arguments, methods, narrative movement, and source hierarchy get compressed into vague takeaways.

Wrong shape

A technical paper, policy report, classic text, and literary essay should not share one outline.

Weak retrieval

Notes often work in the chat but not later in a knowledge base, repository, or local Markdown reader.

Source blur

User-provided material and supplemented scholarship can merge unless the workflow marks them clearly.

03 / 08
Material-Type StructuresDifferent text, different skeleton

It chooses a narrative structure before writing.

Material type Organizing logic What must survive
Research article Problem -> gap -> method -> evidence -> contribution Claims, method, scope, limitations
Review / survey Field map -> traditions -> unsettled questions Taxonomy, debates, research agenda
Classic / canonical text Author's order and textual movement Terms, passages, rhetoric, reception
Literary text Form, voice, imagery, narrative movement How form creates meaning
Case / policy report Situation -> actors -> evidence -> implications Facts, interpretations, limits

Portable tools

The skill now mentions Obsidian, Typora, GitHub/GitLab, and VS Code before drafting.

Source marking

External scholarship and background sources must be approved and clearly labeled.

Material logic

File proof

The generated Markdown file must exist at the intended destination before completion.

Release hygiene

No personal paths, no specialized course residue, MIT license, public GitHub repo.
