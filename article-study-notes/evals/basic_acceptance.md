# Basic Acceptance

## Prompt

Use article-study-notes to turn a philosophy, AI, business, literary, policy, or other substantial text into a Markdown study note.

## Pass Conditions

- Starts with a clear title and reading path.
- Confirms note language and depth before drafting when unspecified.
- Prompts the user that `.md` notes can work well with Obsidian, Typora, GitHub/GitLab, and VS Code, with different strengths.
- Writes a `.md` file by default.
- Reconstructs the source around problem, concepts, argument, scholarship, and understanding points.
- Uses a narrative structure appropriate to the material type rather than forcing every source into the same outline.
- Preserves important definitions, distinctions, argument steps, evidence, examples, limitations, and conclusion boundaries.
- For classical, literary, narrative, or canonical texts, preserves the source's own order, form, narrative movement, imagery, rhetoric, and historical context.
- Does not become a source ledger or paragraph summary.
- Does not collapse unfamiliar or difficult material into a brief abstract.
- Asks before using external scholarly supplementation for complex, broad, or controversial material.
- If external scholarship is used, references are listed and clearly marked.
- If external scholarship is not used, the note says so briefly.
- Does not claim unsupported facts or fabricated citations.
- If Markdown enhancements are requested, they remain restrained and portable.

## Strong Output Signals

- The note is useful to the user's future self.
- The author’s claims and the assistant's scholarly background are distinguishable.
- The note is concise for simple material and information-rich for complex or unfamiliar material.
